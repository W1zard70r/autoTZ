## FUNCTIONAL

# 1. Функциональные Требования

## 1.1. Управление Данными

*   Система должна обеспечивать хранение конфигурационных данных в базе данных (config_storage_requirement).
*   Система должна поддерживать сущность базы данных "User" для представления пользователей (user_entity).
*   Система должна поддерживать сущность базы данных "Order" для представления заказов (order_entity).

## 1.2. Аутентификация и Авторизация

*   Система должна предоставлять эндпоинт `/login` для аутентификации пользователя (login_endpoint).
*   Система должна использовать токен доступа (access token) для аутентифицированных запросов (access_token).
*   Система должна использовать токен обновления (refresh token) для получения нового токена доступа (refresh_token).

## STACK

# 3. Технологический Стек (Tech Stack)

## 3.1. Общие положения
*   В процессе формирования технологического стека было осуществлено обсуждение и принято решение относительно выбора системы управления базами данных.
*   Также проведено обсуждение выбора фронтенд-фреймворка между React и Vue.

## 3.2. База данных
*   В качестве системы управления базами данных выбрана PostgreSQL версии 15.
*   Для обеспечения хранения конфигураций будет задействован тип данных JSONB.
*   Предварительно требуется утверждение схемы базы данных.

## 3.3. Бэкенд
*   Сервис будет реализован с использованием фреймворка FastAPI.
*   Механизм авторизации будет основан на JSON Web Tokens (JWT).

## 3.4. Фронтенд
*   Для разработки пользовательских интерфейсов рассматриваются следующие опции:
    *   React: JavaScript библиотека.
    *   Vue: Прогрессивный JavaScript фреймворк.
*   Сборка фронтенд-проектов будет осуществляться с применением инструмента Vite.
