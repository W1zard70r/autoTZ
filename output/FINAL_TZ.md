# Техническое задание (IT-проект)

## 1. Общее описание проекта
- **Назначение системы:** Система предназначена для обработки и анализа мультимедийного контента с целью извлечения структурированной информации. Она решает проблему автоматического преобразования аудио- и видео-данных в текстовый формат с сохранением метаданных, что позволяет эффективно работать с большими объемами мультимедийных файлов. Система также включает в себя компоненты для создания знаний и построения графов связей между сущностями, что делает её полезной для исследовательских и аналитических задач.
- **Аналоги/конкуренты:** PDF-Extract-Kit - инструмент для извлечения таблиц, полей, чисел и текста из PDF-документов, Аналогичные системы обработки мультимедиа с использованием STT (Speech-to-Text) технологий, RAG (Retrieval-Augmented Generation) системы для обработки и анализа информации
- **Итоговый продукт:** Заказчик получит полностью функциональную систему обработки мультимедийного контента с возможностью извлечения текста и метаданных, построения знаний и графов связей. Также будет предоставлена документация, исходный код, готовый к деплою веб-интерфейс на основе FastAPI и React, а также презентация с демонстрацией возможностей системы.

### Основные сценарии использования
**Загрузка и обработка файла:**
1. Пользователь выбирает файл для загрузки
2. Система проверяет формат и размер файла
3. Файл загружается в S3-хранилище
4. Запускается асинхронная обработка через Celery
5. FFmpeg извлекает аудиодорожку
6. ML-модель выполняет распознавание речи
7. Результаты сохраняются в PostgreSQL
8. Пользователь получает уведомление о завершении
**Просмотр результатов обработки:**
1. Пользователь заходит в личный кабинет
2. Выбирает проект из списка
3. Открывает детальную информацию о файле
4. Просматривает текстовый результат
5. Анализирует метаданные (timestamp, source_extension)
6. Скачивает результат в нужном формате
7. Сравнивает с оригиналом при необходимости
**Создание аналитического отчета:**
1. Аналитик выбирает источник данных
2. Формирует SQL-запрос через Prisma ORM
3. Применяет фильтры и агрегации
4. Создает визуализации с помощью React-компонентов
5. Экспортирует отчет в PDF
6. Настраивает автоматическую отправку
7. Сохраняет шаблон для повторного использования
**Управление проектами:**
1. Администратор создает новый проект
2. Назначает роли участникам
3. Настраивает права доступа
4. Определяет квоты ресурсов

7. Архивирует завершенные проекты

2. Конфигурирует API-эндпоинты

## 4. Технические требования
### Стек технологий
- **Фронтенд:** React — JavaScript библиотека для создания пользовательских интерфейсов с компонентным подходом
- **Бэкенд:** FastAPI — современный веб-фреймворк для Python, SQLAlchemy — ORM для работы с базами данных, Celery — распределённая очередь задач
- **База данных:** PostgreSQL — реляционная СУБД с расширенными возможностями, Redis — in-memory хранилище для кэширования
- Prisma — следующее поколение ORM для Node.js и TypeScript
- OAuth — фреймворк авторизации для безопасного доступа
- Docker — платформа контейнеризации для развёртывания
- aiobotocore — асинхронный клиент AWS S3 для работы с потоками данных
- FFmpeg — инструмент для обработки аудио/видео потоков
- asyncio — библиотека для асинхронного программирования в Python
### Интеграции
- **AWS S3:** Используется для хранения и получения мультимедийных файлов (видео, аудио) через aiobotocore для асинхронной обработки
- **Speech-to-Text API:** Требуется найти бесплатный API для транскрибации аудио в текст, возможно распределить роли между командой для оптимизации затрат
- **PDF-Extract-Kit:** Инструмент для извлечения таблиц, полей, чисел и текста из PDF документов
### Производительность
- Асинхронная обработка аудиопотоков с использованием asyncio и FFmpeg для обработки больших файлов без блокировки
- Параллельная обработка с помощью Celery для распределения нагрузки между воркерами
- Кэширование часто используемых данных в Redis для ускорения доступа
- Оптимизация API для обработки множественных запросов от пользователей

## 5. Нефункциональные требования
### Безопасность
- Использование JWT (JSON Web Tokens) для аутентификации и авторизации пользователей
- HTTPS для шифрования данных при передаче между клиентом и сервером
- OAuth 2.0 для безопасного доступа к внешним сервисам
- Защита API-эндпоинтов с помощью middleware для проверки токенов
- Шифрование конфиденциальных данных в базе данных
- Rate limiting для защиты от DDoS-атак
- Валидация входных данных для предотвращения SQL-инъекций и XSS-атак
### Масштабируемость
- Горизонтальное масштабирование с использованием контейнеров Docker
- Балансировка нагрузки через Nginx или аналогичный прокси
- Использование Redis для кэширования часто запрашиваемых данных
- Асинхронная обработка задач через Celery для разгрузки основного потока
- Оптимизация запросов к PostgreSQL с использованием индексов
- CDN для раздачи статических файлов
- Микросервисная архитектура для независимого масштабирования компонентов
### Время отклика
- API должен отвечать в течение 200-500 мс при нормальной нагрузке
- Загрузка основных страниц приложения не должна превышать 2 секунды
- Асинхронные задачи через Celery должны выполняться в течение 5-10 минут
- Обработка видео/аудио файлов должна занимать не более 3-х раз дольше реального времени
- Кэширование должно снижать latency для часто запрашиваемых данных до 50 мс
